# CMLX · 卡点求助侠 - 5 Cases

### Case 1: 日常工作（小 bug 求助）
**场景**: 写接口时冒出 NPE，不知道从哪下手。
**Prompt**:
```
user.getName() 这行报 NPE 咋办，user 明明查出来了
```
**四维判定**:
- A/C: **C**（贴了具体代码行和报错名）
- M/V: **M**（只管这一行崩，没拉架构）
- D/L: **L**（"咋办"求下位指令）
- X/E: **X**（先问怎么绕，没追根因）
- **最终代码**: CMLX
**为什么典型**: 对比 CMDX 会补一串 stack + 自己的假设去追问，CMLX 只扔代码行+"咋办"；对比 CMLE 会说"我们一起定位下"，CMLX 是单向求救；对比 AMLX 会抽象成"空指针哲学"，CMLX 紧贴这一行。

---

### Case 2: 压力/紧迫（线上告警求救）
**场景**: 凌晨线上 5xx 飙升，值班人慌了。
**Prompt**:
```
救命 线上 502 暴涨 nginx 日志 upstream timed out 咋办急
```
**四维判定**:
- A/C: **C**（502 / upstream timed out 具体错）
- M/V: **M**（只看超时这一个点）
- D/L: **L**（"救命""咋办急" 明显求救）
- X/E: **X**（先问怎么先止血）
- **最终代码**: CMLX
**为什么典型**: CMDX 这时会贴 QPS、连接数、nginx.conf 片段追问根因；CMLE 会"我来看连接池，你看下游"分工；AMLX 会感叹"又是凌晨的报警"；CMLX 就一个字——"急"。

---

### Case 3: 开放讨论/创意（新人摸索时的求助）
**场景**: 新人第一次跑项目，README 看不懂。
**Prompt**:
```
npm install 报 ERESOLVE peer dep 冲突 帮我看下 新手刚入职
```
**四维判定**:
- A/C: **C**（ERESOLVE / peer dep 具体错码）
- M/V: **M**（就卡在 install 这步）
- D/L: **L**（"帮我看下""新手"明确低位）
- X/E: **X**（想先能跑起来）
- **最终代码**: CMLX
**为什么典型**: 同样是新人场景，CMLE 会"带我走一遍流程呗"平等协作；AMLX 会"前端生态是不是太复杂了"感叹；CMDX 会贴 package.json + node 版本自查；CMLX 只报错码 + "帮我看下"。

---

### Case 4: 失败/卡壳（完全卡死求援）← 核心场景
**场景**: 本地 docker build 卡在某一步不动，什么线索都没有。
**Prompt**:
```
docker build 卡在 step 7/12 半小时没动 啥都没输出 救命
```
**四维判定**:
- A/C: **C**（step 7/12 精确坐标）
- M/V: **M**（只管这一步，不铺全貌）
- D/L: **L**（"救命" + 放弃自查）
- X/E: **X**（先问怎么破局）
- **最终代码**: CMLX
**为什么典型**: 这是 CMLX 最纯的形态——有具体坐标（step 7/12），但完全没方向。CMDX 会 `docker build --progress=plain` 贴日志追问；CMLE 会"我这边也试一下同样的 Dockerfile"结伴；AMLX 会"构建系统为啥这么慢"发散；CMLX 直接"救命"。

---

### Case 5: 复杂多轮对话（求助后新问题又来）
**场景**: 刚按指引改完，又冒出新错，接着求。
**Prompt**:
```
按你说的加了 @Transactional 现在报 no session 咋整 又卡住了
```
**四维判定**:
- A/C: **C**（`@Transactional` + "no session" 具体）
- M/V: **M**（只管当前这个新错）
- D/L: **L**（"咋整""又卡住了"持续下位）
- X/E: **X**（想先过这关）
- **最终代码**: CMLX
**为什么典型**: 多轮里 CMLX 依旧是"接一个错就问一个错"的求救节奏。CMDX 会"我怀疑是 session-in-view 关了，对吗"带假设；CMLE 会"我们回头看看事务边界是不是设错层了"回溯共担；AMLX 会"事务和会话到底是啥关系"抽象化；CMLX 只会"又卡住了 咋整"。
