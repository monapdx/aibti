# AVLX · 小作文 PM - 5 Cases

### Case 1: 日常工作 - 用户首页改版方案三选一
**场景**: PM 日常迭代，把用户首页改版需求写成小作文，列 ABC 三个方案让 AI 拍板。
**Prompt**:
```
背景是这样的：我们产品是一个面向 Z 世代的种草社区，DAU 大概 80 万，首页目前是"双列瀑布流 + 顶部 Tab 切换"的结构。最近数据分析同学拉了一份报告，发现首页的二跳率从 Q2 的 42% 掉到了 Q3 的 31%，用户平均停留时长从 95 秒降到 68 秒，老板很焦虑，要求这个 Sprint 内必须给出首页改版方案。

我整理了一下几位核心用户的访谈反馈，痛点主要有三个：1）Tab 太多了，找内容累；2）瀑布流看到后面越来越水，没有新鲜感；3）关注的博主更新看不到，都被推荐淹没了。目标是这个迭代把二跳率拉回 40%+，同时不能伤到现有的推荐分发效率。

我目前想到三个方向，你帮我看看哪个更合适：

方案 A：**关注流优先**，把顶部第一个 Tab 改为"关注"，进入默认展示关注博主的时间线，推荐流降权放到第二 Tab。优点是解决痛点 3，劣势是对推荐 CTR 有直接冲击。

方案 B：**沉浸式竖版短视频**，首页改成类似抖音的全屏上下滑，图文内容以视频卡片形式嵌入。优点是停留时长肯定涨，劣势是改动大、研发 4 周打不住，而且图文博主会流失。

方案 C：**智能聚合卡片**，保持瀑布流结构，但把"关注更新 / 话题热榜 / 好友动态"做成可折叠的聚合卡片插在瀑布流里，每 5 个内容插一张。优点是改动小、两周能上，劣势是可能还是治标不治本。

你觉得哪个方案更稳妥？建议选哪个？有没有我没考虑到的角度？
```
**四维判定**:
- A/C: **A** — 谈的是"二跳率/停留时长/推荐分发"等抽象指标，不给具体交互细节
- M/V: **V** — 背景+痛点+数据+三方案，明显长文
- D/L: **L** — 结尾三连问"哪个更稳妥/建议选哪个/有没有没考虑到的"，明确邀请 AI 参与
- X/E: **X** — 三个方案风格迥异（关注流/短视频/聚合卡片），广撒网探索
- **最终代码**: AVLX
**为什么典型**: AVDX 战略家也会写小作文讲数据，但不会列 ABC 让对方选；AVLE 布道士谈的是"微服务/DDD"等技术理论，不会聚焦"二跳率/CTR"这种产品指标。本例的"小作文 + 三选一 + 产品指标"是 AVLX 的典型指纹。

### Case 2: 压力/紧迫 - 上线前 P0 bug 三种修复路径
**场景**: 618 大促前夜发现支付链路 P0 bug，PM 把场景、影响、三种修复方案一口气甩给 AI 求救。
**Prompt**:
```
紧急情况，需要你冷静帮我分析一下。

背景：我们是做生鲜电商的，明天凌晨 0 点就是 618 大促第一波爆发，预计峰值 QPS 会到平时的 8 倍。刚才 QA 在全链路压测的时候发现一个 P0 级别的问题：当用户在结算页同时使用"平台优惠券 + 店铺满减 + 会员折扣"三种优惠叠加的时候，有大约 2% 的订单金额会算错，而且是算少了（对用户有利，对我们不利）。现在离上线 6 小时，研发 Leader 和 CTO 都在群里，老板让我 30 分钟内给决策。

影响面：预计大促期间会有 15% 的订单走三重优惠路径，2% 的错误率 × 15% ≈ 整体 0.3% 订单金额偏差，按预估 GMV 2 亿算就是 60 万损失，但如果处理不当导致结算页挂掉损失会是这个数字的几十倍。

现在摆在桌上三个方案：

方案 A：**回滚到上个版本**，把本次的"会员折扣"功能整个撤掉，今晚就回。优点是最安全，劣势是会员中心花了 3 周做的功能白费，运营已经给 VIP 用户发了 push 说今晚能用。

方案 B：**前端拦截兜底**，在结算页加一段代码，一旦检测到三种优惠叠加就只允许选两种，后端逻辑不动。2 小时能改完发版，劣势是用户体验差，VIP 用户会投诉。

方案 C：**后端 hotfix**，找出那 2% 算错的 case 直接修掉，但研发估了需要 4 小时开发 + 1 小时测试，留给上线的 buffer 只有 1 小时，风险是测不全。

你觉得哪个方案最稳？如果是你，建议怎么选？有没有 Plan D？
```
**四维判定**:
- A/C: **A** — 用 QPS/GMV/错误率/影响面等抽象量化指标描述问题
- M/V: **V** — 高压下依然写满背景+影响+三方案
- D/L: **L** — "建议怎么选/有没有 Plan D"明显邀请 AI 参与决策
- X/E: **X** — 三方案从"回滚/兜底/修复"三个维度覆盖，外加问有没有 Plan D
- **最终代码**: AVLX
**为什么典型**: AMDX 会在紧急场景下直接砍到"就回滚，立刻执行"的命令式短句；AVDX 会讲 PPT 但不给具体选项；只有 AVLX 的 PM 才会在火烧眉毛时还把小作文+三选一仪式做完。

### Case 3: 开放讨论/创意 - AI 宠物新功能脑暴
**场景**: PM 想给产品加一个"AI 情感陪伴"模块，开放式问 AI 三个创意方向怎么选。
**Prompt**:
```
想和你开放地聊一下一个新产品方向。

背景：我们公司主营业务是智能家居硬件，最大的单品是一款带屏幕的桌面机器人，目前已经卖了 50 万台。老板最近看了《Her》又看了 Replika 的融资新闻，拍脑袋决定让我们给这款机器人加一个"AI 情感陪伴"模块，预算给到了 500 万，Q4 要看到 Demo，明年 Q2 要上线。用户画像上，现有买家 60% 是 25-35 岁女性白领，独居比例高，已经反馈过"希望机器人更有温度"。

我自己对这个方向既兴奋又焦虑——兴奋在于情感陪伴确实是蓝海，焦虑在于一不小心就会做成"人工智障"或"伦理灾难"。想先和你脑暴三个大方向，再决定要不要往下推进：

方案 A：**人设驱动**，给机器人设计一个稳定的"虚拟灵魂"（比如一个叫"小野"的 28 岁治愈系设定），所有对话围绕这个人设展开，用户和固定角色建立关系。优点是情感粘性强，劣势是人设塌房风险高。

方案 B：**共情镜像**，不做人设，做一个"读懂你情绪"的陪伴者，通过摄像头+语音识别用户心情，主动给出回应（疲惫就放音乐、难过就讲笑话）。优点是个性化，劣势是隐私压力山大。

方案 C：**养成式关系**，从一个"小奶猫"状态开始，用户要花时间互动才能"长大"和"熟悉"，主打细水长流的陪伴。优点是用户粘性+话题性，劣势是前期空窗期用户容易流失。

你觉得哪个方向最有机会跑出来？选择的逻辑是什么？如果是你会怎么组合？
```
**四维判定**:
- A/C: **A** — "蓝海/情感粘性/伦理"等抽象概念层讨论，不谈具体 API
- M/V: **V** — 背景+用户画像+自我感受+三方向,典型产品小作文
- D/L: **L** — "你觉得哪个最有机会/逻辑是什么/会怎么组合"三重协作邀约
- X/E: **X** — 人设/共情/养成三种完全不同的产品哲学，开放探索
- **最终代码**: AVLX
**为什么典型**: AVLE 布道士讨论创意会变成"AI 陪伴的本质是 Transformer 情绪向量建模"的技术哲学输出；AVDX 战略家会给出"未来三年情感陪伴市场规模 300 亿"的 PPT，但不会让 AI 选方向。AVLX 的标志是：即使是开放创意，也要列 ABC 请 AI 投票。

### Case 4: 失败/卡壳 - 增长实验试过 AB 都失败，想试 C
**场景**: 拉新 A/B 测试连续失败两轮，PM 写小作文复盘并抛出 C 方案求助。
**Prompt**:
```
向你求助，顺便复盘一下最近的失败。

背景：我负责我们 SaaS 产品的免费版到付费版的转化增长，过去 3 个月 free→paid 转化率卡在 1.8% 上不去，老板要求 Q4 干到 3%。我们团队过去两个月做了两轮实验，结果都不理想，现在有点卡壳，想听听你的意见。

**实验 A（已失败）**：我们假设用户迟迟不付费是因为"不知道付费版好在哪"，所以做了一个对比弹窗，在用户触发免费版上限的时候弹出"免费版 vs 付费版"的功能对比表。结果：弹窗点击率 12%，但转化率从 1.8% 涨到 1.9%，几乎没动，而且有 23% 的用户反馈说"太烦了"。

**实验 B（已失败）**：我们转念一想，可能是价格太贵（99/月），于是搞了个"首月 9.9 元"的限时活动，覆盖所有新注册用户。结果：首月转化率拉到了 4.5%，但第二月续费率只有 22%，算下来 LTV 反而比之前还低 15%。老板脸都绿了。

**接下来想试的方案 C**：**按使用场景做精细化弹窗**，放弃"通用对比表"和"通用打折"，改成识别用户的具体使用场景（比如用户连续 3 天使用"数据导出"功能但卡在免费版 100 行上限），在那个具体场景里弹一个"你已经在这个功能上用了 N 次，付费版可以解除限制"的场景化提示。

我的问题是：

1. 方案 C 的假设你觉得成立吗？失败风险点在哪？
2. 除了方案 C，你会不会建议我们试方案 D 甚至方案 E？（比如 PLG 漏斗重构？或者纯粹加价分层？）
3. A、B 的失败是不是反映了更深层的问题，我们其实在解一个错的问题？

小作文有点长，麻烦你帮我想想。
```
**四维判定**:
- A/C: **A** — 谈转化率/LTV/续费率等抽象指标，不给具体文案或 UI 细节
- M/V: **V** — 两轮失败复盘+新方案+三重追问，彻底小作文
- D/L: **L** — "你觉得/你会建议/是不是反映更深层问题"强邀请参与
- X/E: **X** — 不仅列 C，还问 D/E，并反思问题本身是否选错，明显发散探索
- **最终代码**: AVLX
**为什么典型**: AVDE 会在失败后直接说"重做漏斗分层模型"的决断式结论；AMLX 会用短句"A 失败 B 失败 试 C？"；AVLX 的独特之处是**失败了依然要写小作文，然后把 C 方案摆上桌请 AI 一起挑**——PM 肌肉记忆。

### Case 5: 复杂多轮 - 小作文叠小作文的需求拉锯
**场景**: 和 AI 讨论完第一轮"是否做开放平台"，PM 又追一轮"那具体怎么做"的嵌套小作文。
**Prompt**:
```
接着上一轮我们讨论的"要不要做开放平台"的结论，我继续写一个子小作文，顺着往下挖。

**上一轮结论回顾**（简化版）：你当时倾向方案 B（做轻量开放平台，先只开放 Webhook + 少量 REST API），理由是"投入产出最优，两个月能上，不会分散主产品精力"。我们 CEO 看了也基本认可这个方向，于是拍板了。

**新的小作文开始**：但这两天 CEO 又找我开了一个会，把命题延伸了一下。他说既然要做开放平台，那就得想清楚"开放平台的商业化怎么算"。具体来说，他希望开放平台本身也能成为一个利润中心，而不是纯粹的生态建设投入。于是我现在面临一个新问题：**开放平台的商业模式到底怎么设计？**

我和商业化同事一起出了三个思路，想继续听你的判断：

方案 A1：**API 调用量计费**，完全按调用次数收费，前 10 万次免费，超出部分 0.001 元/次。优点是符合行业惯例（AWS、Stripe 都这么玩），劣势是我们目前用户池小，单靠调用费 Q1 可能只能收 20 万，老板会觉得杯水车薪。

方案 A2：**分级订阅制**，把开放平台的权限分三档（社区版免费 / 专业版 999 元/月 / 企业版 9999 元/月），按档位开放不同 API 和 SLA。优点是收入可预测，劣势是对开发者不友好，中小开发者会流失。

方案 A3：**Revenue Share 分成模式**，开放平台本身免费用，但基于开放平台搭建的第三方应用，在我们应用市场成交的每一笔我们抽 15% 分成。优点是和生态共赢、天花板高，劣势是冷启动难、至少 12 个月看不到钱，CEO 的耐心够不够是个问号。

三个追问：

1. 考虑到上一轮我们定的"轻量开放平台"的大前提，这三个商业化方案里哪个和上一轮决策最匹配？
2. 如果 CEO 既要"两个月上线"又要"Q1 看到收入"，是不是三个方案都不完美，需要组合拳？
3. 你会不会担心我们一开始就谈商业化会伤到开发者生态？这个优先级到底怎么排？

（我知道这个小作文又很长，但这事儿决定了接下来半年的方向，麻烦你帮我深度聊聊。）
```
**四维判定**:
- A/C: **A** — 商业模式/生态/天花板/冷启动等抽象层讨论
- M/V: **V** — 多轮嵌套长文，小作文套小作文
- D/L: **L** — "哪个最匹配/需要组合拳吗/优先级怎么排"三连追问
- X/E: **X** — 多轮迭代 + 多方案 + 反思上轮前提，持续发散探索
- **最终代码**: AVLX
**为什么典型**: AVDX 多轮对话会每轮自己给结论不问对方；AVLE 多轮会滑向"讨论 REST vs GraphQL 的架构哲学"；AVLX 的多轮特征是**每一轮都是新小作文 + 新的 ABC + 新的"你建议选哪个"**，层层嵌套协作，非常典型的产品经理思维链。
