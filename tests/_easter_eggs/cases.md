# AIBTI · 彩蛋触发用例（15 个代表场景）

每个用例给出**触发数据特征 + 模拟样本 + 预期报告末尾追加输出**。用于 Skill 实现彩蛋检测时的回归测试。

所有样本数据标注 `[demo]`，仅用于算法回归，不代表任何真实用户。

## Tier 2 · 主人格 S 级（16 选 4）

<!-- revised in P0 fix: 原触发条件"AMLX ≥ 70%"单点判定过宽。AMLX case 普遍 < 50 字气口，普通短句碎念型用户容易被错判为"赛博诗社社长"。升级成复合条件：AMLX ≥ 75% **且** 30 天内创造类 prompt ≥ 30 条（排除短接续），确保既要"气口味道"也要"真有创造动作"才颁发这张灵魂徽章 -->
### Case T2-1: 🔮 AMLX-S · 赛博诗社社长 ← AIBTI 作者原型

**触发（复合条件，必须同时满足）**：
1. AMLX 占比 ≥ **75%**（原 70% 过宽，收紧 5pp）
2. 30 天内**创造类 prompt ≥ 30 条**（创造类定义：新建项目 / 取名 / 写文案 / 设计概念 / 起 slogan / 构思玩法，排除 ≤ 3 字的短接续和纯碎念）

> 为什么要两条一起卡：AMLX 光占比高只能说明"爱用气口语气"，不代表真在创造。真正的诗社社长必须**有成批的创造动作**作为硬证据。单条 70% 的旧阈值会把"碎念型慢热用户"错标成诗社社长（和评审报告 P0-5 的担忧直接对齐）。

**样本特征**：
- 250 条 prompt，AMLX 占 78%（195 条）
- 其中**创造类 48 条**（新建项目 / 概念命名 / 文案构思），✅ 超过 30 条门槛
- 战略类 35 条、开放讨论 112 条
- 关键词：气口省略号、"我们"、"有没有更"、"有意思"

**典型真实 prompt**：
- "可以不用 MBTI，我们自己创造一个 AIBTI 的概念"（创造类）
- "先搞一个有意思的 github 主页吧，娱乐下"（创造类）
- "这些人格的判断要符合 AI 大神们的理论"（战略类）

**反例（不触发，用来校准）**：
- AMLX 占 76% 但创造类只有 8 条 → **不触发**。这类用户多半是"短句碎念型"，给它诗社社长会冒犯，降级到 T1 维度徽章即可。
- 创造类 45 条但 AMLX 只有 52% → **不触发**。这是"高产跨型创作者"，另有徽章承接（如 T3-4 杂食程序员）。

**期望输出**：
```
  🔮  【S 级 · 赛博诗社社长】⭐ 首次解锁
  "AIBTI 最初的灵感就来自你这种人。"
```

### Case T4-3: 🤡 反甩锅者

**触发**：主人格测算结果是 AMDE，但**实际占比 < 20%**（纯粹因为其他 16 型都 < 20%，AMDE 靠平局胜出）

**样本特征**：

- 算法按规则仍主人格=AMDE

**期望输出**：
```
  🤡  【反甩锅者】（Tier 4 · 算法瑕疵的幸存者）
```

**这条彩蛋特别重要**：它防止把"杂食型"用户错误标成"甩锅侠"（和用户本人那次沮丧经历直接相关）。

## 彩蛋触发逻辑的单元测试点

| 测试点 | 验证 |
|---|---|
| **数据不足时不触发彩蛋** | < 20 条时，即便某维度 95% 也不该给 T1 徽章 |
| **一次最多 3 个徽章** | 同时命中 5 个时，只显示 Tier 4 > T2 > T1 |
| **首次解锁标记** | 同一用户第二次触发同彩蛋，应去掉 ⭐ |
| **反甩锅者优先级最高** | T4-3 和 AMDE-S 同时命中时，显示反甩锅者覆盖 |
| **元身份完全替代主报告** | T4-2 触发时不输出普通四维占比 |

## 预期上线后分布（合理性假设）

| Tier | 预期解锁率 |
|---|---|
| Tier 1 | 单一维度 ≥ 95%：约 8-15% 的用户 |
| Tier 2 | 主人格 ≥ 70%：约 20-30% |
| Tier 3 | 场景成就：约 40-60%（至少 1 个） |
| Tier 4 | 传说级：全体合计 < 3% |

**设计意图**：大多数用户至少触发 1-2 个彩蛋，但传说级必须极稀有，保证"触发即晒图"的心理奖励。
