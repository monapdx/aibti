# CMLE · 结对搭子 - 5 Cases

> 代号：CMLE（Concrete + Minimal + Collaborative + Execute）
> Slogan："边写边聊"
> 特征：短句 + 具体代码 + "咱俩/一起/搭把手" + 要结果

---

### Case 1: 日常工作 - 顺手抽函数
**场景**: 写到一半发现一段逻辑可以抽出来，叫搭子一起改一下。
**Prompt**:
```
来，把这段 parseUser 里的校验抽成 validateUser 函数，咱俩顺手改了。
```
**四维判定**:
- A/C: **C**（给了函数名和具体动作，不是"怎么组织代码"）
- M/V: **M**（一句话，不解释为什么抽）
- D/L: **L**（"来""咱俩""顺手改了"全是搭手语气）
- X/E: **E**（要直接改完，不讨论方案）
- **最终代码**: CMLE

**为什么典型**:
- vs CMDE 机枪手："把 parseUser 里的校验抽成 validateUser" 命令式，没有"咱俩"
- vs CVLE 技术导师：不会先讲"为什么要抽函数，SRP 原则是……"
- vs CMLX 求助侠：不是"这段不会抽，求救"，是平等一起干

### Case 2: 压力紧迫 - 线上 bug 搭手
**场景**: 线上报错，时间紧，叫搭子一块儿看。
**Prompt**:
```
搭把手，线上 order 接口 NPE，看下是不是 user 为空没判，一起改。
```
**四维判定**:
- A/C: **C**（给了接口名、异常类型、怀疑点）
- M/V: **M**（急，没废话）
- D/L: **L**（"搭把手""一起改"完全同事口吻）
- X/E: **E**（现在就动手）
- **最终代码**: CMLE

**为什么典型**:
- vs CMDE："排查 order 接口 NPE，修复空指针" 是单兵指令
- vs CVLE：不会展开讲 NPE 类型、Optional 最佳实践
- vs CMLX：不是"线上炸了救救我"，是"搭把手咱俩一起"

---

### Case 3: 开放讨论 - 小聊后动手
**场景**: 有个写法拿不准，先和搭子商量一下，再动手。
**Prompt**:
```
咱俩看下这段 for 循环，是不是能用 reduce 一把梭？能就改了。
```
**四维判定**:
- A/C: **C**（具体到 for/reduce，不是"怎么写更函数式"）
- M/V: **M**（讨论只有半句）
- D/L: **L**（"咱俩看下""能就改了"，是商量不是命令）
- X/E: **E**（讨论完立刻落地）
- **最终代码**: CMLE

**为什么典型**:
- vs CMDE："把 for 改成 reduce" 无讨论直接下令
- vs CVLE 会展开"reduce vs for 的可读性权衡……"长篇
- vs CMLX：不是"reduce 我不会写，教我"，是对等讨论

---

### Case 4: 失败卡壳 - 同事式求搭手
**场景**: 一个测试挂了查半天没头绪，叫搭子一起看。
**Prompt**:
```
卡住了，testLogin 挂了说 token 对不上，你瞅一眼咱俩一起看看。
```
**四维判定**:
- A/C: **C**（给了测试名、错误信息）
- M/V: **M**（短，没长背景）
- D/L: **L**（"你瞅一眼""咱俩一起看看"是搭手不是求救）
- X/E: **E**（目的是排掉问题）
- **最终代码**: CMLE

**为什么典型**:
- vs CMDE："修复 testLogin 的 token 断言" 是单兵指令
- vs CVLE：不会要"讲讲 JWT 签名机制为什么对不上"
- vs CMLX："我不行了 testLogin 救救" 是单向求救；CMLE 是"咱俩一起看看"，平等

---

### Case 5: 复杂多轮 - 边聊边推进
**场景**: 重构一个接口，和搭子一边写一边聊，分步推。
**Prompt**:
```
咱俩重构下 UserService：先把 save 拆成 create/update 俩方法，改完我再说下一步。
```
**四维判定**:
- A/C: **C**（具体到类名、方法名、拆分方式）
- M/V: **M**（只说第一步，不铺全貌）
- D/L: **L**（"咱俩重构""改完我再说"，持续搭手节奏）
- X/E: **E**（每一步都落到代码）
- **最终代码**: CMLE

**为什么典型**:
- vs CMDE："拆 save 为 create/update" 下完指令就走
- vs CVLE 会先出"重构路线图 + 设计原则 + 风险点"
- vs CMLX：不是"不会重构求带"，而是主导节奏 + 邀请共同推进
