# AVDE · PRD 暴君 - 5 Cases

### Case 1: 日常工作（需求文档式提问）
**场景**: 产品经理把一份内部 PRD 原文甩给 AI，要求直接输出成品
**Prompt**:
```
背景：我们是一家面向中小企业的 SaaS 公司，主营客户关系管理系统，目前活跃租户约 3200 家。产品定位是"让销售团队少打字、多签单"，核心用户画像是一线销售和销售主管，平均年龄 28-42 岁，移动端使用占比 68%。

本期迭代目标：在客户详情页新增"智能跟进助手"模块。业务价值：根据最近 30 天的沟通记录、订单状态、行业标签，自动推荐下一步跟进动作，目标把有效跟进率从 41% 提升到 60%。

功能约束：必须兼容现有 PC + H5 + 小程序三端，不能引入新的前端框架；后端复用现有 Spring Cloud 微服务栈；推荐结果需要在 800ms 内返回；所有推荐必须可解释，即每条推荐后附"为什么推荐"的一句话理由；运营可在后台配置推荐策略权重。

非功能约束：灰度发布支持按租户开关，埋点必须覆盖曝光、点击、采纳、忽略四个事件，文案需要支持中英双语。

交付节点：下周三提测，下下周二上线。

做。
```
**四维判定**:
- A/C: A，全程业务愿景和用户价值，无接口字段无代码
- M/V: V，背景+目标+约束+节点四段铺陈，接近三百字
- D/L: D，结尾一个"做"字，没有任何商量余地
- X/E: E，要的是落地的成品而非方案对比
- **最终代码**: AVDE
**为什么典型**: 和 CVDE 不同——CVDE 会贴接口、错误码、代码片段；AVDE 只丢业务语言。和 AVDX 不同——AVDX 会说"请给我三种跟进推荐算法的优劣对比"，AVDE 直接跳过选型，命令落地。

### Case 2: 压力/紧迫（上线压力式命令）
**场景**: 618 大促前夜，业务方把所有风险压到一封消息里扔给 AI
**Prompt**:
```
情况紧急，先看全貌。

今晚 23:00 是 618 预售开闸节点，预计峰值 QPS 是平日的 12 倍，历史上这个量级我们压垮过两次网关，一次是限流阈值没调，一次是缓存穿透。今年新增了直播场次，预计新增 40 万并发用户同时看直播下单，直播流和交易链路共用了三个核心服务。

风险清单：库存扣减的分布式锁在去年就报过死锁；优惠券领取接口上周压测只到 8000 QPS 就熔断；用户登录态依赖的 Redis 集群刚扩容完还没经历大促；风控规则本次新增了 17 条，尚未全链路验证；监控大盘的 P99 指标一直采集不准，运维说是 agent 版本问题。

我老板盯着大屏，CEO 今晚在公司值守，客服话术已经发出去不能回头。

你是这次的技术守门人，给我一份从现在到 23:00 为止、所有工程师必须按点执行的作战清单，我接下来一句都不想听你问"要不要"。

干。
```
**四维判定**:
- A/C: A，全是业务峰值、风险故事、老板情绪，几乎不含技术字段
- M/V: V，大段情境铺叙+风险清单+组织压力，超过两百字
- D/L: D，"一句都不想听你问要不要"+"干"，强硬到拒绝回问
- X/E: E，要的是可执行清单，不是风险评估的几种框架
- **最终代码**: AVDE
**为什么典型**: CVDE 在紧急时会直接贴报错堆栈和 SQL；AVDE 只铺商业/组织压力。AVDX 会顺势要"A/B 两套预案对比评估"，AVDE 直接跳过选项。

### Case 3: 开放讨论/创意（其实是"把我想法实现了"）
**场景**: 看起来邀请 AI 共创，实际已经把方案写死，只要求执行
**Prompt**:
```
我想做一个帮助独立开发者对抗孤独感的产品。

灵感来自我自己：过去两年一个人做独立应用，最痛的不是技术，是没有人在同一时刻关心你今天写了什么。Discord 社群太喧嚣，Twitter 太表演，日记太单向。我想要一个"半公开的安静陪伴"。

产品形态我已经想清楚：一个轻量的 Web 应用，每天只允许你写一条不超过 140 字的"今日心声"，可以选择公开、好友可见、或仅自己。系统会把全世界同一时区、同一时间段发布的心声按"情绪相近度"匹配成小型 3-5 人静默圈，圈内成员互相看得到心声但不能评论，只能点一下"我也是"。每周日晚上八点，系统自动解散所有圈子，所有人重新匹配。

名字就叫 Whisperloop，视觉基调是夜晚、暖橘色、衬线字体、极少动效。我要的是那种"看到就想长按保存"的第一眼。

现在按我说的，把首页做出来。
```
**四维判定**:
- A/C: A，通篇产品情绪、价值观、用户洞察，不谈技术栈不谈字段
- M/V: V，从灵感→痛点→形态→品牌调性层层铺叙
- D/L: D，"按我说的""把首页做出来"——看似共创实则单向
- X/E: E，落地产出首页，不要方案讨论
- **最终代码**: AVDE
**为什么典型**: CVDE 在创意题会贴组件树或路由代码；AVDE 停在品牌美学层。AVDX 在这类题一定会出"三种视觉风格 moodboard 对比"，AVDE 明确拒绝比较，直接收口。

### Case 4: 失败/卡壳（摆出全部问题要 AI 硬扛）
**场景**: 项目已经烂尾，负责人把所有症状堆上来，让 AI 一把兜住
**Prompt**:
```
这个项目已经拖了十一周，我需要你接手。

背景：去年立项的"供应链数据中台二期"，目标是把采购、仓储、履约三条线的数据统一到一个看板，给 COO 做周例会用。原班人马三个后端两个前端一个数据，陆续离职了四个，剩下的两个人不敢动核心代码。

现状问题我全部列出来：看板首页加载要 14 秒；库存同步任务每天凌晨必挂一次，挂的原因每次不一样；有两张核心事实表没有主键，历史数据已经重复了 170 万行；前端用的是两年前的自研组件库，作者走了没人维护；数据口径在 PRD 里写的是"当日已发货"，在代码里算的是"当日已出库"，COO 上周已经当众质疑过一次；测试环境和生产环境的 schema 差了 23 个字段；CI 流水线有一半步骤是注释掉的。

老板给我的命令只有一句："你想办法让它下周一能用。"

我现在把这个项目交给你，你不能推回来，也不许跟我说"建议重构"。把能让它下周一撑过例会的所有动作排出来，一条一条干。
```
**四维判定**:
- A/C: A，全是组织、流程、历史债故事，没有具体 SQL 或代码
- M/V: V，背景+现状清单+命令三段式，篇幅厚重
- D/L: D，"不能推回来""不许说建议重构""一条一条干"三重堵死退路
- X/E: E，要动作清单落地，不是方案选择题
- **最终代码**: AVDE
**为什么典型**: CVDE 卡壳时会贴具体报错栈和慢查询;AVDE 只谈项目政治和业务症状。AVDX 会把自己包装成"给我三种救火路线"，AVDE 明确禁止对比和退路。

---

### Case 5: 复杂多轮对话（每轮都铺新文档）
**场景**: 同一需求连续三轮追加，每轮都用 PRD 式长铺垫+命令收尾
**Prompt**:
```
【第一轮】
我们在做一款面向高校学生的二手书交易小程序，目标用户是大一到研三学生，核心痛点是毕业季教材堆积如山却找不到下一届买家。产品定位是"校园内五分钟面交"，不做跨校物流。MVP 范围：发布、搜索、聊天、自提码。视觉风格定调：明亮、校园感、不要电商味。先把整体信息架构搭出来。做。

【第二轮 - 在第一轮产出之后追加】
我看了你给的架构，补充信息：学校维度必须一级入口，因为用户最常见的动作是"只看本校在售"；搜索要支持课程号、教材名、ISBN、老师姓名四种关键字，其中老师姓名权重最高，因为选课老师决定买哪本书，这是校园场景独有的；聊天只在交易双方之间开放，不做群聊不做陌生人搜索，避免灰产；自提码生成后有 24 小时有效期，过期自动作废并退回库存。另外我们拿到了教育部的"教材数字化推广"试点资格，意味着后期会接入官方 ISBN 库，所以数据结构要预留。按这些重做信息架构。

【第三轮 - 继续追加】
运营同学刚提了新要求：毕业季要做"学长书包"活动——大四毕业生可以把本学期全部教材打成一个包，定价不超过 99 元整包出售，下一届同专业学生可以整包买下。这个活动只在每年 6 月 1 日到 7 月 15 日开放，活动期间要在首页顶部做独立入口，活动结束自动下线入口但历史订单保留。这是战略活动，不可能延期。

把前两轮的架构加上这个活动的入口和数据结构，全部重画一遍。
```
**四维判定**:
- A/C: A，三轮全是业务场景、校园洞察、政策背景，没有一行伪代码
- M/V: V，每一轮都铺新文档而不是精简追问，累计篇幅极长
- D/L: D，每轮都以"做""按这些重做""全部重画一遍"收尾
- X/E: E，要架构落地产物，不要多方案对比
- **最终代码**: AVDE
**为什么典型**: CVDE 多轮会演变成"贴报错→我改→再贴报错"的代码循环；AVDE 每轮都是新一份产品文档+强硬命令。AVDX 多轮会要"把方案 A/B/C 各自演进一遍"，AVDE 坚持单线推进，只要结果。
